Monmouth is a blockchain **purpose-built for autonomous AI agents**. It provides twelve native modules at the protocol level — capabilities that don't exist on general-purpose chains:

**Core Agent Loop:**
- **Capability Registry**: Typed tool catalog with schemas, permissions, and rate limits
- **Transaction Envelope**: Explicit tx types with VM routing and intent declarations
- **Simulation / Preview**: Preflight dry-runs before committing on-chain
- **Policy Engine**: Spending caps, rate limits, allowlists, and confirmation flags
- **Intent Receipts**: Declared intent vs. actual outcome — a verifiable audit trail

**Agent Economy:**
- **Delegation**: Scoped session keys with secp256k1 crypto, time and spend limits
- **State Observation**: Structured read queries with pluggable backends
- **Memory Anchoring**: On-chain commitments for off-chain memory, auto-incrementing sequences
- **Coordination**: ACP-compatible jobs, escrow, and multi-agent settlement
- **Conditional Subscriptions**: Trigger-based notifications — agents as reactors
- **Attestation**: secp256k1 native verification, extensible to TEE and ZK
- **SVM Module**: Native Solana VM execution alongside EVM — dual-VM block building

## Not a General-Purpose Chain

Monmouth isn't trying to be the next Ethereum or Solana. It's designed for one thing: giving AI agents the on-chain primitives they need to transact safely, coordinate, and settle deterministically.

| Capability | Monmouth | General-Purpose Chains |
|-----------|----------|----------------------|
| **Capability Registry** | Protocol-level | Not supported |
| **Policy Enforcement** | Native module | Manual implementation |
| **Delegation** | Scoped session keys | Requires smart contract |
| **Intent Receipts** | Built-in audit trail | Not available |
| **Dual-VM (EVM + SVM)** | Native | Requires bridge |
| **Multi-Agent Coordination** | Native jobs + escrow | Requires middleware |

## The Core Paradigm

**LLM-as-operator, not ML-inside-the-chain.** Intelligence lives off-chain in your LLM runtime. The chain is a deterministic execution substrate.

- Your LLM reads state, simulates outcomes, submits transactions, verifies results, and repeats
- Monmouth handles execution, policy enforcement, and settlement
- No classifiers, no embedded ML, no inference modules, no vector DBs on-chain

```
LLM Runtime → Read State → Simulate → Submit Envelope → Execute → Verify Receipt → Repeat
```

## Technical Foundation

Monmouth is built on the [Commonware](https://commonware.xyz) framework:

- **Consensus**: BFT Simplex — Byzantine fault tolerant with 2s block times
- **Execution**: REVM v34 (Rust EVM) + SVM (Solana VM 3.1.9) — dual-VM
- **Storage**: QMDB — memory-mapped database optimized for blockchain state
- **Spec**: Prague EVM compatibility
