## JSON-RPC API

Monmouth exposes a standard Ethereum JSON-RPC API with additional agent-specific methods. Any EVM-compatible client works out of the box.

### Endpoint

```
https://rpc.monmouth.io
```

WebSocket:
```
wss://ws.monmouth.io
```

## Standard Ethereum Methods (26)

All standard `eth_*` methods are supported:

| Method | Description |
|--------|-------------|
| `eth_blockNumber` | Current block number |
| `eth_getBalance` | Account balance |
| `eth_getTransactionByHash` | Transaction by hash |
| `eth_getTransactionReceipt` | Transaction receipt |
| `eth_sendRawTransaction` | Submit a signed transaction |
| `eth_call` | Simulate a call (read-only) |
| `eth_estimateGas` | Estimate gas for a transaction |
| `eth_getBlockByNumber` | Block by number |
| `eth_getBlockByHash` | Block by hash |
| `eth_getCode` | Contract bytecode |
| `eth_getStorageAt` | Storage slot value |
| `eth_getTransactionCount` | Account nonce |
| `eth_chainId` | Chain ID (7750) |
| `eth_gasPrice` | Current gas price |
| `eth_getLogs` | Query event logs |
| ... | Full Ethereum JSON-RPC spec |

## Monmouth Methods (7)

### `monmouth_nodeStatus`

Get node status, sync state, and module health.

```json
{
  "jsonrpc": "2.0",
  "method": "monmouth_nodeStatus",
  "params": [],
  "id": 1
}
```

### `monmouth_listCapabilities`

List all registered capabilities in the Capability Registry.

```json
{
  "jsonrpc": "2.0",
  "method": "monmouth_listCapabilities",
  "params": [],
  "id": 1
}
```

Response:
```json
{
  "result": [
    {
      "id": "swap",
      "name": "Token Swap",
      "description": "Swap one token for another",
      "permissionLevel": "public"
    }
  ]
}
```

### `monmouth_getCapability`

Get full details for a specific capability.

```json
{
  "jsonrpc": "2.0",
  "method": "monmouth_getCapability",
  "params": ["swap"],
  "id": 1
}
```

### `monmouth_getCapabilitySchema`

Get the typed JSON schema for a capability's inputs and outputs.

```json
{
  "jsonrpc": "2.0",
  "method": "monmouth_getCapabilitySchema",
  "params": ["swap"],
  "id": 1
}
```

### `monmouth_svmStatus`

Get SVM executor status — account count, last executed block, health.

```json
{
  "jsonrpc": "2.0",
  "method": "monmouth_svmStatus",
  "params": [],
  "id": 1
}
```

### `monmouth_svmGetAccount`

Get a Solana account by its 32-byte public key.

```json
{
  "jsonrpc": "2.0",
  "method": "monmouth_svmGetAccount",
  "params": ["<base58_pubkey>"],
  "id": 1
}
```

Response:
```json
{
  "result": {
    "lamports": 1000000000,
    "owner": "11111111111111111111111111111111",
    "executable": false,
    "data": "base64..."
  }
}
```

### `monmouth_svmGetProgramInfo`

Get metadata about a deployed Solana program.

```json
{
  "jsonrpc": "2.0",
  "method": "monmouth_svmGetProgramInfo",
  "params": ["<base58_program_id>"],
  "id": 1
}
```

## Filter Methods (6)

Polling-based event filters, compatible with standard Ethereum tooling:

| Method | Description |
|--------|-------------|
| `eth_newFilter` | Create a log filter |
| `eth_newBlockFilter` | Create a new-block filter |
| `eth_newPendingTransactionFilter` | Create a pending-tx filter |
| `eth_getFilterChanges` | Poll for new events |
| `eth_getFilterLogs` | Get all logs matching a filter |
| `eth_uninstallFilter` | Remove a filter |

Filter IDs are generated with a cryptographically secure random number generator (OsRng).

## WebSocket Subscriptions

Real-time push notifications via `eth_subscribe` / `eth_unsubscribe`:

### `newHeads`

Subscribe to new block headers:

```json
{
  "jsonrpc": "2.0",
  "method": "eth_subscribe",
  "params": ["newHeads"],
  "id": 1
}
```

### `logs`

Subscribe to contract event logs with optional filter:

```json
{
  "jsonrpc": "2.0",
  "method": "eth_subscribe",
  "params": ["logs", {
    "address": "0x...",
    "topics": ["0x..."]
  }],
  "id": 1
}
```

### `newPendingTransactions`

Subscribe to pending transactions entering the mempool:

```json
{
  "jsonrpc": "2.0",
  "method": "eth_subscribe",
  "params": ["newPendingTransactions"],
  "id": 1
}
```

## Rate Limits

The RPC server enforces per-IP rate limiting with configurable thresholds. JSON-RPC requests are capped at:

- **Request body**: 2 MB max
- **Response body**: 10 MB max
- **Batch requests**: 50 per batch
