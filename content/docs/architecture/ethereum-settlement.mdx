## Why Ethereum?

Monmouth settles to Ethereum for:

- **Security**: Battle-tested consensus with $400B+ securing the network
- **Composability**: Direct access to DeFi protocols
- **Bridging**: Trustless interoperability with other L2s
- **Decentralization**: No reliance on proprietary security models

## Settlement Architecture

```
Monmouth Agent Transactions
         ↓
   Batch Compression
         ↓
   SequencerInbox (L1)
         ↓
   StateCommitmentChain (L1)
         ↓
   Ethereum Consensus
```

## L1 Contracts

### SequencerInbox

Receives compressed batch data from the Monmouth sequencer.

```solidity
interface ISequencerInbox {
    function submitBatch(
        bytes calldata batchData,
        uint256 l2BlockNumber
    ) external;
}
```

### StateCommitmentChain

Stores L2 state roots, enabling verification of Monmouth state on Ethereum.

```solidity
interface IStateCommitmentChain {
    function verifyStateRoot(
        uint256 l2BlockNumber,
        bytes32 stateRoot,
        bytes calldata proof
    ) external view returns (bool);
}
```

### StandardBridge

Handles ETH and ERC-20 deposits/withdrawals between Ethereum and Monmouth.

```solidity
interface IStandardBridge {
    function depositETH(uint256 l2Gas) external payable;
    function depositERC20(
        address l1Token,
        address l2Token,
        uint256 amount,
        uint256 l2Gas
    ) external;

    function finalizeWithdrawal(
        address recipient,
        address token,
        uint256 amount,
        bytes calldata proof
    ) external;
}
```

### CrossDomainMessenger

Arbitrary message passing between Ethereum and Monmouth.

```solidity
interface ICrossDomainMessenger {
    function sendMessage(
        address target,
        bytes calldata message,
        uint256 gasLimit
    ) external;

    function relayMessage(
        address sender,
        address target,
        bytes calldata message,
        uint256 messageNonce
    ) external;
}
```

## Security Model

### Current: Trusted Sequencer

The Monmouth sequencer is currently trusted. This provides:
- Fast finality (~250ms)
- Efficient batching
- Low operational overhead

### Roadmap: Decentralized Sequencing

Future milestones include:
- Based sequencing (L1 proposer ordering)
- Shared sequencing networks
- Proof-of-RAG validator set

## Data Availability

All transaction data is posted to Ethereum L1, ensuring:
- Anyone can reconstruct Monmouth state
- No trusted data availability committee
- Full Ethereum security inheritance
