## Overview

The Message Passer precompile (`0x4200`) handles all communication between Monmouth and Ethereum L1:

- **Deposits**: Move assets from Ethereum to Monmouth
- **Withdrawals**: Move assets from Monmouth to Ethereum
- **Messaging**: Arbitrary data between layers

## Address

```
0x0000000000000000000000000000000000004200
```

## Interface

```solidity
interface IMessagePasser {
    /// @notice Initiate a withdrawal to L1
    /// @param target The L1 address to receive funds/message
    /// @param value The ETH value to send
    /// @param data The calldata to execute on L1
    /// @param gasLimit The L1 gas limit
    function initiateWithdrawal(
        address target,
        uint256 value,
        bytes calldata data,
        uint256 gasLimit
    ) external payable;

    /// @notice Send a message to L1
    /// @param target The L1 contract to call
    /// @param data The message data
    /// @param gasLimit The L1 gas limit
    function sendMessage(
        address target,
        bytes calldata data,
        uint256 gasLimit
    ) external;

    /// @notice Get withdrawal proof for L1 finalization
    /// @param withdrawalHash The withdrawal hash
    /// @return proof The Merkle proof
    function getWithdrawalProof(
        bytes32 withdrawalHash
    ) external view returns (bytes memory proof);

    /// @notice Check if a message has been relayed
    /// @param messageHash The message hash
    /// @return relayed Whether the message was relayed
    function isMessageRelayed(
        bytes32 messageHash
    ) external view returns (bool relayed);
}
```

## Deposits (L1 → Monmouth)

Deposits are initiated on Ethereum L1 via the StandardBridge:

```typescript
// On Ethereum L1
const bridge = new ethers.Contract(
  STANDARD_BRIDGE_ADDRESS,
  StandardBridgeABI,
  signer
)

// Deposit ETH
await bridge.depositETH(200000, { value: parseEther('1') })

// Deposit ERC-20
await bridge.depositERC20(
  l1TokenAddress,
  l2TokenAddress,
  parseUnits('1000', 6), // USDC
  200000
)
```

Deposits appear on Monmouth after L1 confirmation (~12 minutes).

## Withdrawals (Monmouth → L1)

Withdrawals are initiated on Monmouth:

```typescript
import { MessagePasser } from '@monmouth/wallet-sdk'

const passer = new MessagePasser(wallet)

// Initiate withdrawal
const withdrawal = await passer.initiateWithdrawal({
  target: l1RecipientAddress,
  value: parseEther('1'),
  gasLimit: 200000,
})

console.log(withdrawal.hash) // Withdrawal hash
```

After the challenge period, finalize on L1:

```typescript
// Get proof from Monmouth
const proof = await passer.getWithdrawalProof(withdrawal.hash)

// Finalize on L1
const l1Bridge = new ethers.Contract(
  STANDARD_BRIDGE_ADDRESS,
  StandardBridgeABI,
  l1Signer
)

await l1Bridge.finalizeWithdrawal(
  withdrawal.target,
  withdrawal.token,
  withdrawal.value,
  proof
)
```

## Cross-Layer Messaging

Send arbitrary messages between layers:

```typescript
// Send message to L1 contract
await passer.sendMessage({
  target: l1ContractAddress,
  data: l1Contract.interface.encodeFunctionData('someFunction', [arg1, arg2]),
  gasLimit: 200000,
})
```

Receive messages from L1:

```solidity
// On Monmouth
contract MyContract {
    address public l1Messenger;

    modifier onlyL1Messenger() {
        require(msg.sender == address(0x4200), "Not from L1");
        _;
    }

    function receiveMessage(bytes calldata data) external onlyL1Messenger {
        // Process L1 message
    }
}
```

## SDK Helpers

The Wallet SDK provides high-level helpers:

```typescript
// Bridge ETH to Monmouth
await wallet.bridge({
  direction: 'deposit',
  token: 'ETH',
  amount: '1 ETH',
})

// Bridge back to Ethereum
await wallet.bridge({
  direction: 'withdraw',
  token: 'ETH',
  amount: '1 ETH',
  recipient: '0x...', // Optional, defaults to agent
})
```

## Timing

| Operation | Duration |
|-----------|----------|
| Deposit confirmation | ~12 minutes (L1 finality) |
| Withdrawal initiation | Instant on Monmouth |
| Challenge period | 7 days (mainnet) / 1 hour (testnet) |
| Withdrawal finalization | ~12 minutes (L1 finality) |
